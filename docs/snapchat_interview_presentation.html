Covariance P15×15Error state

Why Error-State?

  • Quaternion has 4 DOF, only 3 physical → over-parameterised
  • Error Δθ ∈ ℝ³ via SO(3) exponential map
  • No quaternion renormalisation needed
  • Covariance is minimal (15×15, not 16×16)
// Boxplus update (orientation)
q ← expSO3(δθ) ⊗ q
p ← p + δp
v ← v + δv
Sophus SE3d handles all Lie group operations (exp, log, adjoint) so quaternion normalisation, gimbal lock, and singularities are non-issues throughout the codebase.

IMU Predict — RK4 + Error Propagation

Continuous-Time Kinematics

ṗ = v
v̇ = R·(ameas−b_a) + gw
q̇ = q ⊗ [0, (ωmeas−b_g)/2]
ḃ_g = 0, ḃ_a = 0

F Matrix (15×15 error-state)

F = [ 0 I 0 0 0 ] ← δṗ
    [ 0 0 −R[ac]× 0 −R ] ← δv̇
    [ 0 0 −[ωc]× −I 0 ] ← δθ̇
    [ 0 0 0 0 0 ] ← δḃ_g
    [ 0 0 0 0 0 ] ← δḃ_a

Covariance Propagation

Φ = I + F·Δt
P ← Φ P Φᵀ + G·Qc·Gᵀ·Δt

Optimisation: Precomputed G·Qc·Gᵀ

Every block of G·Qc·Gᵀ simplifies to σ²·I because G contains ±R or ±I slices and R·Rᵀ = I. The entire 15×15 product is rotation-independent and constant — computed once at construction, saving a 15×12 matrix build + triple product on every one of 200 IMU calls/sec.
RK4 vs Euler: At 200 Hz, Euler integration error is O(dt²) ≈ 2.5×10⁻⁵. RK4 reduces this to O(dt⁴) ≈ 6×10⁻¹⁰ — critical for head-shake transients.

Pre-Built LK Pyramid

  • buildOpticalFlowPyramid() called once at frame end
  • Next frame reuses it — eliminates one pyramid build per frame
  • Also removes a full pixel-buffer deep copy

Persistent Feature IDs

  • Each feature carries an integer ID across frames
  • EKF associates landmarks without re-triangulating every frame
  • Enables landmark age management (evict stale after N frames)

Stereo Matching Challenge

Diagnostic finding: LK stereo converges for 95% of features, but 93% of those converge to wrong local minima off the epipolar line.

lk_ok=286/300 · epipolar_fail=265 · final_ok=21
Root cause: Initialised at right_pts = left_pts (zero-disparity guess). With large disparity, LK locks onto nearby texture off the epipolar line.

Fix: For tracked features, initialise from prev_right + left_motion_delta.

EKF Visual Update — Measurement Model

Measurement h(p_c) — 4 residuals per feature

// p_c = [X, Y, Z] in left-cam frame
u_l = fx·X/Z + cx
v_l = fy·Y/Z + cy
u_r = fx·(X−b)/Z + cx // baseline b
v_r = fy·Y/Z + cy // = v_l (rectified)

Kalman Update — Joseph Form

S_k = H_k · P · H_kᵀ + R_i // 4×4
K_k = P · H_kᵀ · S_k⁻¹ // 15×4
IKH = I − K_k · H_k
P = IKH·P·IKHᵀ + K_k·R_i·K_kᵀ // numerically stable

Gating — Two Layers

  • Pixel gate: reject if |residual|∞ > 40 px — protects against inflated P during initialisation
  • Mahalanobis gate: χ² test on 4-DOF residual — removes statistical outliers

Landmark Management

  • 3D positions stored in a map keyed by feature ID
  • New features triangulated and added to landmark map
  • Landmarks evicted after landmark_max_age frames without observation
Landmark reuse across frames: the EKF sees the same 3D point across many frames, building strong geometric constraints for each landmark.

Bottleneck Discovery — Profiling First

Initial Profile — EuRoC V1_01_easy

Stagems/frameShare
Stereo rectification0.552%
Feature tracking1.445%
EKF update25.4693%
Total27.4536 Hz
Surprise: The feature tracker (assumed slow) took 1.4 ms. The EKF matrix solve (assumed cheap) took 25 ms. Measurement before optimising revealed where the time actually went.

Root Cause — O((4N)³) Batch Solve

// Stack all N features into one matrix
H_all : (4N × 15), N ≈ 200
R_mat : (4N × 4N) diagonal

S = H_all · P · H_allᵀ + R_mat
// → 800×800 matrix

LDLT(S).solve(...) // ~512M FLOP per frame
At N = 200 features and 20 Hz camera rate, ~10 billion FLOP/sec spent on a single linear algebra step. No amount of LK tuning would have fixed this.
09

The Fix — Sequential Kalman Update

Key Insight

For independent measurement noise (each feature's reprojection error is drawn independently), batch and sequential Kalman updates are mathematically equivalent.

Sequential Loop

for feature k = 0 … N−1:
  S_k = H_k·P·H_kᵀ + R_i // 4×4 Cholesky
  K_k = P·H_kᵀ·S_k⁻¹ // 15×4
  state ⊞= K_k · residual_k // apply now
  IKH = I − K_k·H_k
  P = IKH·P·IKHᵀ + K_k·R_i·K_kᵀ
Cost: N × ~10,000 FLOP = 2M FLOP for N=200
vs 512M FLOP batch → ~250× reduction

Why Sequential is Also More Accurate

  • Implicit down-weighting: after feature k, P is tightened — later redundant features get smaller Kalman gains automatically
  • Better conditioning: each 4×4 Cholesky is inherently well-conditioned; the 800×800 batch matrix suffers from near-collinear features
  • Better linearisation: each Jacobian is evaluated against a progressively-corrected state, reducing nonlinear error
Each 4×4 Cholesky is inherently well-conditioned; large batch matrices suffer from near-collinear features. Sequential update also progressively tightens P, so later redundant features get smaller gains automatically.

What This Means for Embedded Deployment

  • 128 MB RSS, perfectly flat — no allocations in the hot path; all Eigen matrices stack-allocated or pre-allocated. Zero memory growth over the session confirms no leaks.
  • 24% avg on one core — the tracking + EKF loop leaves ≥ 75% of one CPU core free for a render thread on the same SoC.
  • Burst to 83% — FAST detection spikes are bounded and infrequent; the system never saturates a core end-to-end.
  • Rerun visualization (disabled): < 0.2% total — async producer/consumer queue completely decouples I/O from the realtime path.

Key Takeaways

"The bottleneck is always where you don't expect it."
Profiling showed the feature tracker (assumed slow) took 1.4 ms/frame, while the EKF matrix solve (assumed cheap) took 25 ms/frame. A single algorithmic change delivered a 98× speedup with no loss in accuracy.
510 Hz headroom
Leaves ample budget for rendering, reprojection, and pose prediction on embedded SoC.
Predictable latency
Sequential Kalman gives bounded, fixed-time-per-feature updates — no latency spikes from large matrix solves.
Ready to extend
Modular C++20 architecture: lazy mapping thread, and camera models can be added as separate modules.
